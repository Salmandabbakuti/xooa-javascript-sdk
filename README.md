### basePath

The base URL against which to resolve every API call's (relative) path.

Type: [String][429]

### logger

Setting Logger
t

### token

Token

### defaultHeaders

The default HTTP headers to be included for all API calls.

Type: [Array][430]&lt;[String][429]>

### timeout

The default HTTP timeout for all API calls.

Type: [Number][431]

### cache

If set to false an additional timestamp parameter is added to all API GET calls to
prevent browser caching

Type: [Boolean][432]

### enableCookies

If set to true, the client will save the cookies from each server
response, and return them in the next request.

### setLoggerLevel

Returns a string representation for an actual parameter.

#### Parameters

-   `level`  The actual parameter.

Returns **[String][429]** The string representation of <code>param</code>.

### buildUrl

Builds full URL by appending the given path to the base URL and replacing path parameter place-holders with parameter values.
NOTE: query parameters are not handled here.

#### Parameters

-   `path` **[String][429]** The path to append to the base URL.
-   `pathParams` **[Object][433]** The parameter values to append.

Returns **[String][429]** The encoded path with parameter values substituted.

### isJsonMime

Checks whether the given content type represents JSON.<br>
JSON content type examples:<br>

<ul>
<li>application/json</li>
<li>application/json; charset=UTF8</li>
<li>APPLICATION/JSON</li>
</ul>

#### Parameters

-   `contentType` **[String][429]** The MIME content type to check.

Returns **[Boolean][432]** <code>true</code> if <code>contentType</code> represents JSON, otherwise <code>false</code>.

### jsonPreferredMime

Chooses a content type from the given array, with JSON preferred; i.e. return JSON if included, otherwise return the first.

#### Parameters

-   `contentTypes` **[Array][430]&lt;[String][429]>** 

Returns **[String][429]** The chosen content type, preferring JSON.

### isFileParam

Checks whether the given parameter value represents file-like content.

#### Parameters

-   `param`  The parameter to check.

Returns **[Boolean][432]** <code>true</code> if <code>param</code> represents a file.

### normalizeParams

Normalizes parameter values:

<ul>
<li>remove nils</li>
<li>keep files and arrays</li>
<li>format to string with `paramToString` for other cases</li>
</ul>

#### Parameters

-   `params` **[Object][433]&lt;[String][429], [Object][433]>** The parameters as object properties.

Returns **[Object][433]&lt;[String][429], [Object][433]>** normalized parameters.

### buildCollectionParam

Builds a string representation of an array-type actual parameter, according to the given collection format.

#### Parameters

-   `param` **[Array][430]** An array parameter.
-   `collectionFormat` **module:ApiClient.CollectionFormatEnum** The array element separator strategy.

Returns **([String][429] \| [Array][430])** A string representation of the supplied collection, using the specified delimiter. Returns
<code>param</code> as is if <code>collectionFormat</code> is <code>multi</code>.

### applyAuthToRequest

Applies authentication headers to the request.

#### Parameters

-   `request` **[Object][433]** The request object created by a <code>superagent()</code> call.
-   `authNames` **[Array][430]&lt;[String][429]>** An array of authentication method names.

### deserialize

Deserializes an HTTP response body into a value of the specified type.

#### Parameters

-   `response` **[Object][433]** A SuperAgent response object.
-   `returnType` **([String][429] \| [Array][430]&lt;[String][429]> | [Object][433]&lt;[String][429], [Object][433]> | [Function][434])** The type to return. Pass a string for simple types
    or the constructor function for a complex type. Pass an array containing the type name to return an array of that type. To
    return an object, pass an object with one property whose name is the key type and whose value is the corresponding value type:
    all properties on <code>data<code> will be converted to this type.

Returns **any** A value of the specified type.

### callApi

Invokes the REST service using the supplied settings and parameters.

#### Parameters

-   `path` **[String][429]** The base URL to invoke.
-   `httpMethod` **[String][429]** The HTTP method to use.
-   `pathParams` **[Object][433]&lt;[String][429], [String][429]>** A map of path parameters and their values.
-   `queryParams` **[Object][433]&lt;[String][429], [Object][433]>** A map of query parameters and their values.
-   `collectionQueryParams` **[Object][433]&lt;[String][429], [Object][433]>** A map of collection query parameters and their values.
-   `headerParams` **[Object][433]&lt;[String][429], [Object][433]>** A map of header parameters and their values.
-   `formParams` **[Object][433]&lt;[String][429], [Object][433]>** A map of form parameters and their values.
-   `bodyParam` **[Object][433]** The value to pass as the request body.
-   `contentTypes` **[Array][430]&lt;[String][429]>** An array of request MIME types.
-   `accepts` **[Array][430]&lt;[String][429]>** An array of acceptable response MIME types.
-   `returnType` **([String][429] \| [Array][430] | ObjectFunction)** The required type to return; can be a string for simple types or the
    constructor for a complex type.
-   `async` **[Boolean][432]** Async or Sync call
-   `callback` **[module:ApiClient~callApiCallback][435]** The callback function.
-   `authNames` **[Array][430]&lt;[String][429]>** An array of authentication type names.

Returns **[Object][433]** The SuperAgent request object.

### CollectionFormatEnum

Enumeration of collection format separator strategies.

#### CSV

Comma-separated values. Value: <code>csv</code>

#### SSV

Space-separated values. Value: <code>ssv</code>

#### TSV

Tab-separated values. Value: <code>tsv</code>

#### PIPES

Pipe(|)-separated values. Value: <code>pipes</code>

#### MULTI

Native array. Value: <code>multi</code>

### parseDate

Parses an ISO-8601 string representation of a date value.

#### Parameters

-   `str` **[String][429]** The date value as a string.

Returns **[Date][436]** The parsed date object.

### convertToType

Converts a value to the specified type.

#### Parameters

-   `data` **([String][429] \| [Object][433])** The data to convert, as a string or object.
-   `type` **([String][429] \| [Array][430]&lt;[String][429]> | [Object][433]&lt;[String][429], [Object][433]> | [Function][434])** The type to return. Pass a string for simple types
    or the constructor function for a complex type. Pass an array containing the type name to return an array of that type. To
    return an object, pass an object with one property whose name is the key type and whose value is the corresponding value type:
    all properties on <code>data<code> will be converted to this type.

Returns **any** An instance of the specified type or null or undefined if data is null or undefined.

### constructFromObject

Constructs a new map or array model from REST data.

#### Parameters

-   `data`  {Object|Array} The REST data.
-   `obj`  {Object|Array} The target object or array.
-   `itemType`

Type: [module:ApiClient][437]

## query

Query Blockchain data
The query API End Point is used for querying blockchain state. The end point must call a function already defined in your Smart Contract app which will process the query request. The function name is part of the endpoint URL, or can be entered as the fcn parameter  when testing using the Sandbox. The function arguments (number of arguments and type) is determined by the Smart Contract. The Smart Contract is responsible for validation and exception management. For example, if testing the sample get-set Smart Contract app, enter get (without quotes) as the value for fcn.   The response body is also determined by the Smart Contract app, and thats also the reason why a consistent response sample is unavailable for this end point. A success response may be either 200 or 202. For more details refer to Synchronous vs Asynchronous Calls. In contrast to Invoke, the Query end point will often return fast even when called in Synchronous mode  Required permission: read (\\"Access\\":\\"rw\\" or \\"Access\\":\\"r\\")

### invoke

Submit blockchain transaction
The Invoke API End Point is used for submitting transaction for processing by the blockchain Smart Contract app. The end point must call a function already defined in your Smart Contract app which will process the Invoke request. The function name is part of the endpoint URL, or can be entered as the fcn parameter  when testing using the Sandbox. For example, if testing the sample get-set Smart Contract app, use set (without quotes)  as the value for fcn.   The function arguments (number of arguments and type) is determined by the Smart Contract. The Smart Contract is also responsible for arguments validation and exception management. The payload object of Invoke Transaction Response in case of Final Response is determined by the Smart Contract app.   A success response may be either 200 or 202. For more details refer to Synchronous vs Asynchronous Calls Required permission: write (\\"Access\\":\\"rw\\")

Query Blockchain data
The query API End Point is used for querying blockchain state. The end point must call a function already defined in your Smart Contract app which will process the query request.
